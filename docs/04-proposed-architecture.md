## レイヤー詳細

### 1. 分散コンピューティング基盤

- **Ray/Dask** — 分散タスクスケジューリングとアクターモデル
- **vLLM/Ollama** — LLM推論サービング

### 2. 人口基盤層

- **US**: ACS PUMSから州×年齢層×性別×所得五分位で層化サブサンプリング
- **JP**: e-Statクロス表からIPF（反復比例当てはめ法）で合成マイクロデータを生成

### 3. LLMアーキタイプサービス層

- 200-1,000のアーキタイプにクラスタリング
- バッチLLMクエリでコスト最適化
- AgentTorch方式のアーキタイプ単位推論

### 4. コアシミュレーションエンジン（4モジュール）

| モジュール | 責務 |
| --- | --- |
| エージェントマネージャー | BDI+JTBD状態管理、意思決定ロジック |
| 市場エンジン | 価格・競合・需給・在庫のダイナミクス |
| チャネルシミュレータ | Bass拡散モデル、バイラル係数、マーケティング施策効果 |
| ソーシャルネットワーク | 小世界+スケールフリーネットワーク、意見リーダー、口コミ伝播 |

### 5. シミュレーションオーケストレーション層

- 時間エンジン（タイムステップ管理）
- イベントスケジューラー（マーケティングキャンペーン、競合アクション等の外部イベント注入）
- シナリオ管理（A/Bテスト、感度分析）

### 6. 分析ダッシュボード・API層

- 採用曲線（S字カーブ）のリアルタイム可視化
- 市場シェア推移
- チャネルROI分析
- 感度分析（パラメータ変動による結果変化）

## 技術的課題と緩和策

### 1. JTBDの計算モデル化（難度：高）

先行実装が皆無。ODI指標のエージェント意思決定への統合は新規の設計課題。

### 2. 日本の人口統計データ（難度：中）

IPF合成人口生成パイプラインの構築で対処可能だが、e-Statのクロス表の粒度が合成品質を決定する。

### 3. LLMエージェントの行動一貫性（難度：高）

すべてのプロジェクトで課題として認識されている。
- `temperature=0` + `seed`パラメータの使用
- アーキタイプレベルでの検証
